# Built-in Extensions

## Overview

Axiomkit provides a rich ecosystem of built-in extensions that enable seamless integration with various platforms and services. These extensions bundle related contexts, actions, and services to provide ready-to-use functionality for common use cases.

## Available Extensions

### 🔧 **CLI Extension** (`@axiomkit/cli`)
Command-line interface for terminal-based interactions.

**Features:**
- Interactive terminal interface
- Real-time conversation handling
- Configurable instructions and behavior
- Built-in readline support

**Installation:**
```bash
pnpm add @axiomkit/cli
```

**Usage:**
```typescript
import { cliExtension } from "@axiomkit/cli";

const agent = createAgent({
  model: groq("gemma2-9b-it"),
  extensions: [cliExtension],
});
```

### 🗄️ **MongoDB Extension** (`@axiomkit/mongodb`)
MongoDB integration for persistent memory storage.

**Features:**
- Document-based memory storage
- Automatic indexing
- Connection pooling
- Backup and restore capabilities
- Aggregation pipeline support

**Installation:**
```bash
pnpm add @axiomkit/mongodb mongodb
```

**Usage:**
```typescript
import { MongoDBProvider } from "@axiomkit/mongodb";

const memory = new MemorySystem({
  providers: {
    episodic: new MongoDBProvider({
      connectionString: process.env.MONGODB_URI,
      database: "agent_memory",
      collection: "episodes"
    }),
  },
});
```

## Extension Configuration

### Environment Variables

Each extension requires specific environment variables for configuration:

```bash
# Discord
DISCORD_TOKEN=your_discord_bot_token
DISCORD_CLIENT_ID=your_client_id

# Telegram
TELEGRAM_BOT_TOKEN=your_telegram_bot_token

# MongoDB
MONGODB_URI=mongodb://localhost:27017/agent_memory

# Supabase
SUPABASE_URL=https://your-project.supabase.co
SUPABASE_KEY=your_supabase_anon_key

# SEI
SEI_RPC_URL=https://sei-rpc-endpoint
WALLET_PRIVATE_KEY=your_wallet_private_key
```

### Extension Options

Most extensions support configuration options:

```typescript
// CLI Extension with custom configuration
const customCliExtension = createCliExtension({
  name: "custom-assistant",
  instructions: [
    "You are a specialized assistant for technical support.",
    "Provide detailed, step-by-step solutions.",
    "Always ask for clarification when needed."
  ],
  maxSteps: 10,
  schema: z.object({
    user: z.string(),
    priority: z.enum(["low", "medium", "high"]).optional(),
  }),
});

// Discord Extension with custom settings
const discordExtension = discord({
  intents: [
    GatewayIntentBits.Guilds,
    GatewayIntentBits.GuildMessages,
    GatewayIntentBits.MessageContent,
  ],
  prefix: "!",
  allowedChannels: ["bot-commands", "general"],
});
```

## Creating Custom Extensions

You can create custom extensions by bundling related functionality:

```typescript
import { extension, context, action, service } from "@axiomkit/core";

// Custom context
const customContext = context({
  type: "custom-domain",
  schema: z.object({
    domain: z.string(),
  }),
  key: ({ domain }) => domain,
  create: (state) => ({
    data: [],
    metadata: {},
  }),
  render: ({ memory }) => `Domain: ${memory.data.length} items`,
});

// Custom action
const customAction = action({
  name: "custom-operation",
  description: "Perform a custom operation",
  schema: z.object({
    input: z.string(),
  }),
  handler: async (args, ctx, agent) => {
    // Custom logic here
    return { result: "success" };
  },
});

// Custom service
const customService = service({
  register(container) {
    container.singleton("customService", () => new CustomService());
  },
});

// Bundle into extension
export const customExtension = extension({
  name: "custom",
  contexts: {
    custom: customContext,
  },
  actions: [customAction],
  services: [customService],
});
```

## Extension Best Practices

### 1. **Modular Design**
```typescript
// Good: Separate concerns
const discordContext = context({ /* context config */ });
const discordActions = [action1, action2, action3];
const discordServices = [service1, service2];

export const discordExtension = extension({
  name: "discord",
  contexts: { discord: discordContext },
  actions: discordActions,
  services: discordServices,
});

// Avoid: Monolithic extension
export const discordExtension = extension({
  name: "discord",
  // Everything mixed together
});
```

### 2. **Error Handling**
```typescript
// Good: Comprehensive error handling
const robustAction = action({
  name: "robust-operation",
  handler: async (args, ctx, agent) => {
    try {
      const result = await externalService.call(args);
      return { success: true, data: result };
    } catch (error) {
      // Log error for debugging
      console.error("Operation failed:", error);
      
      // Return structured error response
      return {
        success: false,
        error: error.message,
        retryable: error.retryable || false,
      };
    }
  },
});
```

### 3. **Configuration Management**
```typescript
// Good: Flexible configuration
interface ExtensionConfig {
  enabled: boolean;
  options: Record<string, any>;
  limits: {
    maxRequests: number;
    timeout: number;
  };
}

const configurableExtension = (config: ExtensionConfig) => extension({
  name: "configurable",
  // Use config in extension setup
});
```

### 4. **Testing Extensions**
```typescript
// Test extension functionality
describe("Custom Extension", () => {
  it("should handle context creation", async () => {
    const agent = createAgent({
      model: mockModel,
      extensions: [customExtension],
    });
    
    const result = await agent.run({
      context: customContext,
      args: { domain: "test" },
      input: "test input",
    });
    
    expect(result).toBeDefined();
  });
});
```
