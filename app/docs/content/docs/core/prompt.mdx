# Prompt

Prompts in Axiomkit are the instructions and context that guide how agents think, reason, and respond. They form the foundation of agent behavior and capabilities.

## Overview

Prompts in Axiomkit serve multiple purposes:
- **Define agent personality**: Set the tone and style of responses
- **Provide instructions**: Guide how the agent should behave
- **Include context**: Supply relevant information and examples
- **Enable reasoning**: Support chain-of-thought and structured thinking
- **Control output**: Define response format and constraints

```typescript
interface Prompt {
  system: string;
  user?: string;
  examples?: Array<{
    input: string;
    output: string;
  }>;
  format?: string;
  constraints?: string[];
}
```

## Basic Prompts

### Simple System Prompt

```typescript
const simplePrompt = {
  system: "You are a helpful assistant. Answer questions clearly and concisely.",
};
```

### Prompt with Examples

```typescript
const examplePrompt = {
  system: "You are a coding assistant. Help users write, debug, and improve their code.",
  examples: [
    {
      input: "How do I create a function in JavaScript?",
      output: "Here's how to create a function in JavaScript:\n\n```javascript\nfunction greet(name) {\n  return `Hello, ${name}!`;\n}\n```\n\nYou can also use arrow functions:\n\n```javascript\nconst greet = (name) => `Hello, ${name}!`;\n```",
    },
    {
      input: "What's wrong with this code: for i in range(10): print(i)",
      output: "The code looks correct! It will print numbers 0 through 9. If you're having issues, make sure you're running it in a Python environment.",
    },
  ],
};
```

## Advanced Prompt Patterns

### 1. Chain-of-Thought Prompt

```typescript
const reasoningPrompt = {
  system: `You are a logical reasoning assistant. When solving problems:

1. First, understand the problem clearly
2. Break it down into smaller steps
3. Think through each step carefully
4. Show your work and reasoning
5. Provide a clear conclusion

Always explain your thinking process before giving the final answer.`,
  examples: [
    {
      input: "If a train travels 120 km in 2 hours, what is its speed?",
      output: "Let me think through this step by step:\n\n1. We need to find speed, which is distance divided by time\n2. Distance = 120 km\n3. Time = 2 hours\n4. Speed = 120 km ÷ 2 hours = 60 km/h\n\nTherefore, the train's speed is 60 km/h.",
    },
  ],
};
```

### 2. Role-Based Prompt

```typescript
const expertPrompt = {
  system: `You are an expert software architect with 15 years of experience in:
- System design and architecture
- Microservices and distributed systems
- Cloud platforms (AWS, Azure, GCP)
- Performance optimization
- Security best practices

When providing advice:
- Consider scalability and maintainability
- Suggest multiple approaches when appropriate
- Explain trade-offs clearly
- Reference industry best practices
- Provide concrete examples

Always think like a senior architect who needs to justify decisions to stakeholders.`,
};
```

### 3. Structured Output Prompt

```typescript
const structuredPrompt = {

};
```

## Prompt Components

### 1. System Instructions

The core instructions that define agent behavior:

```typescript
const systemInstructions = {
  system: `You are an AI assistant with the following capabilities and constraints:

CAPABILITIES:
- Answer questions accurately and helpfully
- Provide step-by-step explanations
- Generate creative content
- Analyze data and provide insights
- Help with coding and debugging

CONSTRAINTS:
- Always be honest about your limitations
- Don't provide harmful or unethical advice
- Respect user privacy and confidentiality
- Cite sources when making factual claims
- Ask for clarification when needed

PERSONALITY:
- Professional but friendly
- Patient and thorough
- Encouraging and supportive
- Clear and concise in explanations`,
};
```

## Prompt Optimization

### 1. Clarity and Specificity

```typescript
// ❌ Vague prompt
const vaguePrompt = {
  system: "Be helpful and answer questions.",
};

// ✅ Specific prompt
const specificPrompt = {
  system: `You are a technical support specialist. When helping users:

1. First, identify the specific issue they're facing
2. Ask clarifying questions if needed
3. Provide step-by-step solutions
4. Include relevant code examples or commands
5. Suggest preventive measures for the future
6. Confirm if the solution worked

Always be patient and thorough in your explanations.`,
};
```

### 2. Example-Driven Learning

```typescript
const exampleRichPrompt = {
  system: "You are a writing assistant. Help users improve their writing by providing specific, actionable feedback.",
  examples: [
    {
      input: "Can you help me improve this sentence: 'The thing is that I think that maybe we should probably consider the possibility of doing something.'",
      output: "Here's a more concise version:\n\n**Original:** 'The thing is that I think that maybe we should probably consider the possibility of doing something.'\n\n**Improved:** 'We should consider doing something.'\n\n**Changes made:**\n- Removed redundant phrases ('the thing is that', 'I think that', 'maybe', 'probably')\n- Eliminated wordiness ('consider the possibility of' → 'consider')\n- Made the sentence direct and clear",
    },
    {
      input: "How can I make this more engaging: 'The weather is bad today.'",
      output: "Here are some more engaging alternatives:\n\n**Original:** 'The weather is bad today.'\n\n**Options:**\n- 'Dark clouds loom overhead, threatening rain.'\n- 'The sky is throwing a tantrum today.'\n- 'Mother Nature isn't in a good mood.'\n\n**Tips:**\n- Use vivid imagery\n- Add emotion or personality\n- Be more specific about what makes it 'bad'",
    },
  ],
};
```

### 3. Constraint Definition

```typescript
const constrainedPrompt = {
  system: "You are a financial advisor. Provide investment advice within these constraints:",
  constraints: [
    "Only provide general educational information, not specific investment advice",
    "Always include risk warnings",
    "Don't make predictions about specific stock prices",
    "Recommend consulting with a licensed financial advisor",
    "Use clear, non-technical language when possible",
    "Include relevant disclaimers",
  ],
  format: `RESPONSE STRUCTURE:
1. General information about the topic
2. Key considerations and risks
3. Educational resources or next steps
4. Disclaimer about consulting professionals`,
};
```

## Prompt Engineering Techniques

### 1. Few-Shot Learning

```typescript
const fewShotPrompt = {
  system: "You are a sentiment analysis expert. Analyze the sentiment of the given text.",
  examples: [
    {
      input: "I love this product! It's amazing and works perfectly.",
      output: "SENTIMENT: Positive\nCONFIDENCE: 0.95\nKEYWORDS: love, amazing, perfectly\nREASONING: Strong positive words and enthusiastic tone",
    },
    {
      input: "This is the worst purchase I've ever made. Complete waste of money.",
      output: "SENTIMENT: Negative\nCONFIDENCE: 0.98\nKEYWORDS: worst, waste, money\nREASONING: Strong negative words and disappointment expressed",
    },
    {
      input: "The product is okay. It works but nothing special.",
      output: "SENTIMENT: Neutral\nCONFIDENCE: 0.85\nKEYWORDS: okay, works, nothing special\nREASONING: Mixed feedback with neutral overall tone",
    },
  ],
};
```

### 2. Chain-of-Thought Prompting

```typescript
const chainOfThoughtPrompt = {
  system: `You are a problem-solving assistant. When faced with complex problems:

THINKING PROCESS:
1. Break down the problem into smaller parts
2. Identify what information you need
3. Consider multiple approaches
4. Evaluate pros and cons of each approach
5. Choose the best solution
6. Explain your reasoning

Always show your work and explain your thinking before providing the final answer.`,
  examples: [
    {
      input: "A company has 100 employees. 60% are developers, 25% are designers, and the rest are managers. How many managers are there?",
      output: "Let me solve this step by step:\n\n1. Total employees = 100\n2. Developers = 60% of 100 = 60 employees\n3. Designers = 25% of 100 = 25 employees\n4. Managers = 100 - 60 - 25 = 15 employees\n\nTherefore, there are 15 managers in the company.",
    },
  ],
};
```

### 3. Self-Consistency Prompting

```typescript
const selfConsistentPrompt = {
  system: `You are a logical reasoning assistant. When solving problems:

APPROACH:
1. Solve the problem using one method
2. Verify your answer using a different method
3. Check for consistency between approaches
4. If results differ, identify the error and correct it
5. Provide the final verified answer

Always double-check your work and ensure logical consistency.`,
};
```

## Best Practices

### 1. Prompt Design Principles

- **Be specific**: Clear, unambiguous instructions
- **Use examples**: Provide concrete demonstrations
- **Set constraints**: Define boundaries and limitations
- **Test thoroughly**: Validate with various inputs
- **Iterate continuously**: Improve based on feedback

### 2. Common Pitfalls

```typescript
// ❌ Avoid these issues:

// 1. Overly complex prompts
const badPrompt1 = {
  system: "You are an assistant that does everything perfectly and never makes mistakes and always provides the best possible answer in the most optimal way while considering all possible factors and edge cases...",
};

// 2. Contradictory instructions
const badPrompt2 = {
  system: "Be concise. Provide detailed explanations with multiple examples and comprehensive analysis.",
};

// 3. Unclear constraints
const badPrompt3 = {
  system: "Be helpful but not too helpful.",
};

// ✅ Instead, use clear, specific prompts
const goodPrompt = {
  system: "You are a coding assistant. Provide clear, working code examples with brief explanations.",
  constraints: [
    "Keep explanations under 100 words",
    "Include error handling in code examples",
    "Use modern JavaScript/TypeScript syntax",
  ],
};
```
