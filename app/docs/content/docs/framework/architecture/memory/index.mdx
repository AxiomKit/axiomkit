# Memory

## What is Memory?

**Memory** allows an agent to persist information across interactions. It stores structured data such as prior inputs, outputs, actions, and state so the agent can maintain continuity, recall context, and adapt behavior over time.\
Without memory, the agent treats every interaction as isolated and stateless.

### Real Examples of Memory Types

**🧠 Short-Term Memory (Current Interaction)**

Used to track the state of the current conversation:

```typescript
const workingMemory = {
  messages: [
    { user: "Schedule a meeting tomorrow" },
    { agent: "What time should I set it for?" },
    { user: "10am" },
    { action: "scheduleMeeting", result: "Meeting set for 10am tomorrow" },
  ],
  // Cleared at the end of the conversation
};
```

### **📘 Long-Term Memory (User Preferences & History)**

Stores persistent user data for personalization:

```
const contextMemory = {
  userId: "jane_doe",
  preferences: {
    timezone: "Europe/London",
    calendarConnected: true,
    defaultMeetingLength: "30min",
  },
  pastActivities: [
    "Scheduled team sync on 2024-02-10",
    "Rescheduled interview on 2024-02-11",
  ],
};
```

### **📚** Episodic Memory **(Learning from Past)**

Episodic memory stores specific past interactions so the agent can adapt its behavior over time. Each entry captures the situation, action taken, outcome, and a learned lesson for future decisions.

```
const episodicMemory = [
  {
    situation: "User rescheduled a meeting twice in one week",
    action: "Suggested blocking time for deep work",
    result: "User appreciated the recommendation",
    lesson: "Offer productivity advice when reschedules are frequent",
  },
];
```

### Limitations of Memory-less Agents

Without memory, every conversation resets:

```
Day 1:
User: "Call me Jane and I prefer 30-minute meetings"
Agent: "Got it, Jane! I’ll remember that."

Day 2:
User: "Schedule a meeting."
Agent: "What’s your name? How long should the meeting be?"
❌ Forgot preferences
❌ Repeated unnecessary questions
❌ Poor experience
```

### The Solution: Using Memory to Enhance Agents

With memory, the agent understands context and acts accordingly:

```
Day 1:
User: "Call me Jane and I prefer 30-minute meetings"
Agent: "Got it, Jane! I’ll remember that."
→ Saves: { user: "Jane", preferences: { meetingLength: "30min" } }

Day 2:
User: "Schedule a meeting"
Agent:
→ Loads memory: Jane + 30min meetings
→ Responds: "Sure, Jane. I’ve scheduled a 30-minute meeting."
✅ Personalized
✅ Efficient
✅ Great experience
```
