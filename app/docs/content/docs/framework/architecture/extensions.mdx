# Extensions

## Overview

Extensions are the primary way to add new capabilities to your Axiomkit agents. They bundle related functionality into reusable modules that can be easily shared and distributed. Extensions can include:

- **Actions** - Functions that agents can execute
- **Contexts** - Specialized conversation contexts
- **Memory Providers** - Custom storage backends
- **Platform Integrations** - Connections to external services
- **Custom Logic** - Domain-specific behaviors

## Built-in Extensions

Axiomkit comes with several built-in extensions that provide common functionality:

### 📱 Telegram Extension

Connect your agent to Telegram for messaging capabilities.

```typescript
import { createTelegramExtension } from "@axiomkit/telegram";

const telegramExt = createTelegramExtension({
  botToken: process.env.TELEGRAM_BOT_TOKEN,
  commands: {
    "/start": "Welcome to your AI assistant!",
    "/help": "Available commands: /start, /help, /chat",
    "/chat": "Start a conversation with the AI"
  },
  actions: [sendEmail, createTicket, searchDatabase],
  onMessage: async (message, agent) => {
    // Custom message handling
    const result = await agent.run({
      context: { type: "telegram", userId: message.from.id },
      args: { message: message.text }
    });
    return result.output;
  }
});
```

**Use Cases:**
- Customer support bots
- Notification systems
- Interactive assistants
- Community management

### 🎮 Discord Extension

Integrate with Discord for gaming and community applications.

```typescript
import { createDiscordExtension } from "@axiomkit/discord";

const discordExt = createDiscordExtension({
  token: process.env.DISCORD_BOT_TOKEN,
  intents: ["GuildMessages", "DirectMessages"],
  commands: {
    "!help": "Show available commands",
    "!chat": "Start a conversation",
    "!game": "Play a game"
  },
  actions: [gameActions, moderationActions],
  onMessage: async (message, agent) => {
    // Handle Discord messages
    if (message.content.startsWith("!")) {
      const result = await agent.run({
        context: { type: "discord", channelId: message.channelId },
        args: { message: message.content }
      });
      await message.reply(result.output);
    }
  }
});
```

**Use Cases:**
- Gaming bots
- Community moderation
- Role management
- Event coordination

## Creating Custom Extensions

### Basic Extension Structure

```typescript
import { createExtension } from "@axiomkit/core";

const myExtension = createExtension({
  name: "my-custom-extension",
  version: "1.0.0",
  description: "A custom extension for specific functionality",
  
  // Define actions
  actions: [
    customAction1,
    customAction2
  ],
  
  // Define contexts
  contexts: [
    customContext1,
    customContext2
  ],
  
  // Lifecycle hooks
  onStart: async (agent) => {
    console.log("Extension started");
    // Initialize resources
  },
  
  onStop: async (agent) => {
    console.log("Extension stopped");
    // Cleanup resources
  },
  
  // Configuration schema
  configSchema: z.object({
    apiKey: z.string(),
    endpoint: z.string().url(),
    timeout: z.number().default(5000)
  })
});
```

### Creating Actions

Actions define what your agent can do:

```typescript
import { createAction } from "@axiomkit/core";
import { z } from "zod";

const sendNotification = createAction({
  name: "send_notification",
  description: "Send a notification to a user or channel",
  parameters: {
    recipient: { 
      type: "string", 
      description: "User ID or channel name" 
    },
    message: { 
      type: "string", 
      description: "Notification message" 
    },
    priority: { 
      type: "string", 
      enum: ["low", "medium", "high"],
      default: "medium",
      description: "Notification priority level"
    }
  },
  execute: async ({ recipient, message, priority }) => {
    // Implementation
    const result = await notificationService.send({
      to: recipient,
      message,
      priority
    });
    
    return {
      success: true,
      notificationId: result.id,
      sentAt: new Date().toISOString()
    };
  }
});
```

### Creating Contexts

Contexts define specialized conversation modes:

```typescript
const supportContext = {
  type: "customer_support",
  description: "Context for customer support interactions",
  instructions: [
    "You are a customer support agent",
    "Always be helpful and professional",
    "Escalate complex issues to human agents",
    "Use the customer's name when available"
  ],
  schema: z.object({
    customerId: z.string(),
    ticketId: z.string().optional(),
    priority: z.enum(["low", "medium", "high"]).default("medium")
  }),
  setup: async (args, settings, agent) => {
    // Load customer history
    const customerHistory = await agent.memory.query({
      question: `Customer ${args.customerId} interaction history`,
      limit: 10
    });
    
    return {
      customerId: args.customerId,
      ticketId: args.ticketId,
      priority: args.priority,
      history: customerHistory,
      sessionStart: new Date().toISOString()
    };
  }
};
```

### Extension Configuration

Extensions can accept configuration:

```typescript
const configurableExtension = createExtension({
  name: "configurable-extension",
  version: "1.0.0",
  
  // Define configuration schema
  configSchema: z.object({
    apiKey: z.string(),
    baseUrl: z.string().url(),
    timeout: z.number().min(1000).max(30000).default(5000),
    retries: z.number().min(0).max(5).default(3)
  }),
  
  // Initialize with config
  onStart: async (agent, config) => {
    console.log(`Extension started with base URL: ${config.baseUrl}`);
    
    // Initialize API client with config
    apiClient.init({
      apiKey: config.apiKey,
      baseUrl: config.baseUrl,
      timeout: config.timeout,
      retries: config.retries
    });
  },
  
  actions: [apiAction1, apiAction2]
});

// Use with configuration
const agent = createAgent({
  model: groq("deepseek-r1-distill-llama-70b"),
  extensions: [
    configurableExtension.configure({
      apiKey: process.env.API_KEY,
      baseUrl: "https://api.example.com",
      timeout: 10000,
      retries: 3
    })
  ]
});
```

## Extension Lifecycle

### 1. Registration

Extensions are registered when the agent starts:

```typescript
// Extensions are loaded and validated
await agent.start();
```

### 2. Initialization

Each extension's `onStart` method is called:

```typescript
onStart: async (agent, config) => {
  // Initialize resources
  // Set up event listeners
  // Connect to external services
}
```

### 3. Runtime

Extensions provide actions and contexts during agent operation:

```typescript
// Actions are available to the agent
const result = await agent.run({
  context: { type: "custom_context" },
  args: { action: "custom_action", data: "..." }
});
```

### 4. Cleanup

Extensions are cleaned up when the agent stops:

```typescript
onStop: async (agent) => {
  // Close connections
  // Clean up resources
  // Save state
}
```

## Best Practices

### 1. Extension Design

- **Single Responsibility** - Each extension should have a clear, focused purpose
- **Modularity** - Break complex functionality into smaller, composable extensions
- **Reusability** - Design extensions to be used across different projects
- **Configuration** - Make extensions configurable for different environments

### 2. Action Design

- **Clear Names** - Use descriptive action names that explain their purpose
- **Type Safety** - Leverage TypeScript for compile-time safety
- **Error Handling** - Always handle potential failures gracefully
- **Documentation** - Provide clear descriptions and examples

### 3. Context Design

- **Focused Instructions** - Keep context instructions clear and specific
- **Relevant Schema** - Only include fields that are actually needed
- **Proper Setup** - Use setup functions to initialize context-specific data
- **Memory Integration** - Leverage the memory system for context persistence

### 4. Testing

```typescript
// Test your extension
import { createTestAgent } from "@axiomkit/core/testing";

const testAgent = createTestAgent({
  extensions: [myExtension]
});

test("extension should handle action correctly", async () => {
  const result = await testAgent.run({
    context: { type: "test_context" },
    args: { action: "test_action", data: "test" }
  });
  
  expect(result.output).toBeDefined();
});
```

### Package Structure

```

├── package.json
├── src/
```

## Examples

### Weather Extension

```typescript
import { createExtension, createAction } from "@axiomkit/core";

const weatherAction = createAction({
  name: "get_weather",
  description: "Get current weather for a location",
  parameters: {
    location: { type: "string", description: "City name or coordinates" },
    units: { type: "string", enum: ["metric", "imperial"], default: "metric" }
  },
  execute: async ({ location, units }) => {
    const weather = await weatherAPI.getCurrent(location, units);
    return {
      location,
      temperature: weather.temp,
      condition: weather.condition,
      humidity: weather.humidity
    };
  }
});

export const weatherExtension = createExtension({
  name: "weather",
  version: "1.0.0",
  description: "Weather information and forecasts",
  actions: [weatherAction]
});
```

### Analytics Extension

```typescript
import { createExtension, createAction } from "@axiomkit/core";

const trackEvent = createAction({
  name: "track_event",
  description: "Track an analytics event",
  parameters: {
    event: { type: "string", description: "Event name" },
    properties: { type: "object", description: "Event properties" }
  },
  execute: async ({ event, properties }) => {
    await analytics.track(event, properties);
    return { success: true, eventId: generateId() };
  }
});

export const analyticsExtension = createExtension({
  name: "analytics",
  version: "1.0.0",
  description: "Analytics and tracking capabilities",
  actions: [trackEvent],
  onStart: async (agent, config) => {
    analytics.init(config.apiKey);
  }
});
```
