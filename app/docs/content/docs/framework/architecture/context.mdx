# Context

Contexts in Axiomkit provide a way to define the environment, instructions, and state for your agents. They act as a blueprint for how the agent should behave and what information it should have access to.

## Overview

A context defines:
- **Instructions**: How the agent should behave
- **Schema**: What data the context expects
- **Setup**: How to initialize the context
- **Examples**: Sample interactions for the agent to learn from

```typescript
interface Context<TMemory, Schema, Ctx, Actions, Events> {
  type: string;
  schema: z.ZodSchema<Schema>;
  setup?: (args: Schema, settings: any, agent: Agent) => Promise<Ctx>;
  instructions: string;
  examples?: Array<{
    input: string;
    output: string;
  }>;
}
```

## Basic Context

### Simple Chat Context

```typescript
import { z } from "zod";

const chatContext = {
  type: "chat",
  schema: z.object({
    message: z.string(),
    userId: z.string().optional(),
  }),
  instructions: "You are a helpful assistant. Respond to user messages clearly and concisely.",
};
```

### Using the Context

```typescript
const result = await agent.run({
  context: chatContext,
  args: {
    message: "Hello, how are you?",
    userId: "user123",
  },
});
```

## Advanced Context Patterns

### 1. Context with Setup

```typescript
const userProfileContext = {
  type: "user_profile",
  schema: z.object({
    userId: z.string(),
    sessionId: z.string(),
    preferences: z.object({
      language: z.enum(["en", "es", "fr"]).default("en"),
      theme: z.enum(["light", "dark"]).default("light"),
    }).optional(),
  }),
  setup: async (args, settings, agent) => {
    // Load user profile from database
    const userProfile = await loadUserProfile(args.userId);
    
    // Get user's conversation history
    const history = await agent.memory.getUserEpisodes(args.userId, 10);
    
    return {
      profile: userProfile,
      history: history,
      sessionStart: new Date().toISOString(),
      preferences: args.preferences || userProfile.preferences,
    };
  },
  instructions: "You are a personalized assistant. Use the user's profile and history to provide relevant responses.",
};
```

### 2. Context with Examples

```typescript
const codingAssistantContext = {
  type: "coding_assistant",
  schema: z.object({
    language: z.enum(["javascript", "python", "typescript", "go"]),
    task: z.string(),
    code: z.string().optional(),
  }),
  instructions: "You are a coding assistant. Help users write, debug, and improve their code.",
  examples: [
    {
      input: "Help me write a function to calculate fibonacci numbers in JavaScript",
      output: "Here's a JavaScript function to calculate Fibonacci numbers:\n\n```javascript\nfunction fibonacci(n) {\n  if (n <= 1) return n;\n  return fibonacci(n - 1) + fibonacci(n - 2);\n}\n```\n\nThis recursive function calculates the nth Fibonacci number.",
    },
    {
      input: "Debug this Python code: for i in range(10): print(i)",
      output: "The code looks correct! It will print numbers 0 through 9. If you're having issues, make sure you're running it in a Python environment.",
    },
  ],
};
```

### 3. Multi-Step Context

```typescript
const projectPlanningContext = {
  type: "project_planning",
  schema: z.object({
    projectName: z.string(),
    requirements: z.array(z.string()),
    timeline: z.number().optional(), // days
    teamSize: z.number().optional(),
  }),
  setup: async (args, settings, agent) => {
    // Load similar projects for reference
    const similarProjects = await findSimilarProjects(args.projectName);
    
    // Calculate estimated complexity
    const complexity = calculateComplexity(args.requirements, args.teamSize);
    
    return {
      project: {
        name: args.projectName,
        requirements: args.requirements,
        timeline: args.timeline,
        teamSize: args.teamSize,
      },
      similarProjects: similarProjects,
      complexity: complexity,
      planningPhase: "initial",
    };
  },
  instructions: "You are a project planning assistant. Help users break down projects into manageable tasks and create realistic timelines.",
};
```

## Context Types

### 1. Chat Contexts

For conversational interactions:

```typescript
const customerSupportContext = {
  type: "customer_support",
  schema: z.object({
    userId: z.string(),
    issue: z.string(),
    priority: z.enum(["low", "medium", "high", "urgent"]).default("medium"),
    category: z.enum(["technical", "billing", "general"]).optional(),
  }),
  setup: async (args, settings, agent) => {
    const userHistory = await agent.memory.getUserEpisodes(args.userId, 5);
    const similarIssues = await findSimilarIssues(args.issue);
    
    return {
      userHistory: userHistory,
      similarIssues: similarIssues,
      escalationLevel: determineEscalationLevel(args.priority),
    };
  },
  instructions: "You are a customer support agent. Be empathetic, helpful, and provide clear solutions to user issues.",
};
```

### 2. Task Contexts

For specific task execution:

```typescript
const dataAnalysisContext = {
  type: "data_analysis",
  schema: z.object({
    dataset: z.string(),
    analysisType: z.enum(["descriptive", "predictive", "diagnostic"]),
    metrics: z.array(z.string()).optional(),
  }),
  setup: async (args, settings, agent) => {
    const dataset = await loadDataset(args.dataset);
    const availableMetrics = await getAvailableMetrics(dataset);
    
    return {
      dataset: dataset,
      availableMetrics: availableMetrics,
      analysisType: args.analysisType,
      selectedMetrics: args.metrics || availableMetrics.slice(0, 5),
    };
  },
  instructions: "You are a data analyst. Help users understand their data and provide insights based on the analysis type.",
};
```

### 3. Learning Contexts

For educational interactions:

```typescript
const tutorContext = {
  type: "tutor",
  schema: z.object({
    subject: z.string(),
    level: z.enum(["beginner", "intermediate", "advanced"]),
    studentId: z.string(),
    topic: z.string(),
  }),
  setup: async (args, settings, agent) => {
    const studentProgress = await getStudentProgress(args.studentId, args.subject);
    const topicResources = await getTopicResources(args.topic, args.level);
    
    return {
      studentProgress: studentProgress,
      topicResources: topicResources,
      difficulty: args.level,
      learningStyle: studentProgress.learningStyle || "visual",
    };
  },
  instructions: "You are a patient tutor. Adapt your teaching style to the student's level and learning preferences.",
};
```

## Context Composition

### Combining Multiple Contexts

```typescript
const comprehensiveContext = {
  type: "comprehensive_assistant",
  schema: z.object({
    userId: z.string(),
    sessionId: z.string(),
    task: z.string(),
    preferences: z.object({
      language: z.string().default("en"),
      detailLevel: z.enum(["brief", "detailed", "comprehensive"]).default("detailed"),
    }).optional(),
  }),
  setup: async (args, settings, agent) => {
    // Combine multiple context setups
    const userProfile = await loadUserProfile(args.userId);
    const sessionHistory = await agent.memory.getSessionEpisodes(args.sessionId);
    const taskContext = await analyzeTask(args.task);
    
    return {
      user: userProfile,
      session: {
        id: args.sessionId,
        history: sessionHistory,
        startTime: new Date().toISOString(),
      },
      task: taskContext,
      preferences: args.preferences || userProfile.defaultPreferences,
    };
  },
  instructions: "You are a comprehensive assistant. Use all available context to provide the best possible response.",
};
```

## Context Validation

### Schema Validation

```typescript
import { z } from "zod";

const validatedContext = {
  type: "validated_context",
  schema: z.object({
    email: z.string().email(),
    age: z.number().min(13).max(120),
    interests: z.array(z.string()).min(1).max(10),
    notificationPreferences: z.object({
      email: z.boolean().default(true),
      push: z.boolean().default(false),
      sms: z.boolean().default(false),
    }).optional(),
  }),
  setup: async (args, settings, agent) => {
    // Schema validation happens automatically
    // Additional business logic validation
    if (args.age < 18) {
      throw new Error("Must be 18 or older to use this service");
    }
    
    return {
      validatedUser: args,
      accessLevel: determineAccessLevel(args.age, args.interests),
    };
  },
  instructions: "You are a validated assistant. Only provide appropriate content based on user validation.",
};
```

## Best Practices

### 1. Context Design

- **Keep contexts focused**: Each context should have a specific purpose
- **Use descriptive types**: Make context types meaningful and self-documenting
- **Validate inputs**: Always use proper schema validation
- **Handle errors gracefully**: Provide fallbacks for missing data

### 2. Performance Considerations

```typescript
const optimizedContext = {
  type: "optimized_context",
  schema: z.object({
    userId: z.string(),
    // Only include necessary fields
  }),
  setup: async (args, settings, agent) => {
    // Use caching for expensive operations
    const cacheKey = `user_${args.userId}`;
    let userData = await cache.get(cacheKey);
    
    if (!userData) {
      userData = await loadUserData(args.userId);
      await cache.set(cacheKey, userData, 300); // Cache for 5 minutes
    }
    
    return { userData };
  },
  instructions: "You are an optimized assistant.",
};
```

### 3. Security Considerations

```typescript
const secureContext = {
  type: "secure_context",
  schema: z.object({
    userId: z.string(),
    sessionToken: z.string(),
  }),
  setup: async (args, settings, agent) => {
    // Validate session token
    const isValid = await validateSessionToken(args.sessionToken, args.userId);
    if (!isValid) {
      throw new Error("Invalid session token");
    }
    
    // Sanitize user data
    const userData = await loadUserData(args.userId);
    const sanitizedData = sanitizeUserData(userData);
    
    return { userData: sanitizedData };
  },
  instructions: "You are a secure assistant. Never expose sensitive information.",
};
```

## Common Patterns

### 1. Context Factory

```typescript
function createUserContext(userType: "admin" | "user" | "guest") {
  return {
    type: `user_${userType}`,
    schema: z.object({
      userId: z.string(),
      sessionId: z.string(),
    }),
    setup: async (args, settings, agent) => {
      const userData = await loadUserData(args.userId);
      
      switch (userType) {
        case "admin":
          return { userData, permissions: ["read", "write", "admin"] };
        case "user":
          return { userData, permissions: ["read", "write"] };
        case "guest":
          return { userData, permissions: ["read"] };
      }
    },
    instructions: `You are a ${userType} assistant. Respect user permissions.`,
  };
}

// Usage
const adminContext = createUserContext("admin");
const userContext = createUserContext("user");
```

### 2. Context Composition

```typescript
function composeContexts(...contexts: Context[]) {
  return {
    type: "composed",
    schema: z.intersection(...contexts.map(c => c.schema)),
    setup: async (args, settings, agent) => {
      const results = await Promise.all(
        contexts.map(c => c.setup?.(args, settings, agent) || {})
      );
      return Object.assign({}, ...results);
    },
    instructions: contexts.map(c => c.instructions).join(" "),
  };
}
```
