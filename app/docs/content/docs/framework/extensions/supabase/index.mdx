# Supabase Integration

The `@axiomkit/supabase` package provides a complete memory system implementation backed by Supabase, enabling AI agents to store and retrieve data using Supabase's PostgreSQL database with vector embeddings, key-value storage, and graph relationships.

## Overview

The Supabase integration offers three main memory providers:

- **Key-Value Storage**: Simple key-value pairs with TTL support
- **Vector Storage**: Semantic search using pgvector embeddings
- **Graph Storage**: Relationship-based memory using nodes and edges

All providers are built on top of Supabase's PostgreSQL database, leveraging the power of SQL with the convenience of a managed backend.

## Quick Start

### Installation

```bash
npm install @axiomkit/supabase
# or
pnpm add @axiomkit/supabase
```

### Basic Setup

```typescript
import { createSupabaseMemory } from "@axiomkit/supabase";

// Create a complete memory system
const memory = createSupabaseMemory({
  url: "https://your-project.supabase.co",
  key: "your-supabase-anon-key",
});

// Initialize the memory system
await memory.initialize();

// Use with an agent
const agent = new Agent({
  memory,
  // ... other config
});
```

## Configuration

### SupabaseMemoryConfig

```typescript
interface SupabaseMemoryConfig {
  /** Supabase project URL */
  url: string;
  /** Supabase API key (anon key recommended) */
  key: string;
  /** Table name for key-value storage (default: "kv_store") */
  kvTableName?: string;
  /** Table name for vector embeddings (default: "vector_store") */
  vectorTableName?: string;
  /** Table name for graph nodes (default: "graph_nodes") */
  nodesTableName?: string;
  /** Table name for graph edges (default: "graph_edges") */
  edgesTableName?: string;
  /** Embedding dimension for vectors (default: 1536) */
  embeddingDimension?: number;
}
```

### Database Setup

Before using the Supabase integration, you need to set up your database tables. Run the following SQL in your Supabase SQL editor:

```sql
-- Enable the pgvector extension
CREATE EXTENSION IF NOT EXISTS vector;

-- Key-value store table
CREATE TABLE IF NOT EXISTS kv_store (
  key_hash TEXT PRIMARY KEY,
  key TEXT NOT NULL,
  value TEXT NOT NULL,
  expires_at TIMESTAMP WITH TIME ZONE,
  created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
  updated_at TIMESTAMP WITH TIME ZONE DEFAULT NOW()
);

-- Vector store table
CREATE TABLE IF NOT EXISTS vector_store (
  id TEXT PRIMARY KEY,
  content TEXT NOT NULL,
  embedding VECTOR(1536),
  metadata JSONB,
  created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW()
);

-- Graph nodes table
CREATE TABLE IF NOT EXISTS graph_nodes (
  id TEXT PRIMARY KEY,
  type TEXT NOT NULL,
  properties JSONB,
  labels TEXT[],
  created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
  updated_at TIMESTAMP WITH TIME ZONE DEFAULT NOW()
);

-- Graph edges table
CREATE TABLE IF NOT EXISTS graph_edges (
  id TEXT PRIMARY KEY,
  source_id TEXT NOT NULL REFERENCES graph_nodes(id) ON DELETE CASCADE,
  target_id TEXT NOT NULL REFERENCES graph_nodes(id) ON DELETE CASCADE,
  type TEXT NOT NULL,
  properties JSONB,
  created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
  UNIQUE(source_id, target_id, type)
);

-- Indexes for better performance
CREATE INDEX IF NOT EXISTS idx_kv_store_expires ON kv_store(expires_at);
CREATE INDEX IF NOT EXISTS idx_vector_store_embedding ON vector_store USING ivfflat (embedding vector_cosine_ops);
CREATE INDEX IF NOT EXISTS idx_graph_edges_source ON graph_edges(source_id);
CREATE INDEX IF NOT EXISTS idx_graph_edges_target ON graph_edges(target_id);
```

## Memory Providers

### Key-Value Provider

The Key-Value provider offers simple storage with automatic expiration:

```typescript
import { createSupabaseKVProvider } from "@axiomkit/supabase";

const kvProvider = createSupabaseKVProvider({
  url: "https://your-project.supabase.co",
  key: "your-supabase-anon-key",
  tableName: "kv_store", // optional
});

await kvProvider.initialize();

// Store a value
await kvProvider.set("user:123:preferences", { theme: "dark", language: "en" });

// Store with expiration (1 hour)
await kvProvider.set("session:456", sessionData, { ttl: 3600 });

// Retrieve a value
const preferences = await kvProvider.get("user:123:preferences");

// Delete a value
await kvProvider.delete("session:456");

// Check health
const health = await kvProvider.health();
```

### Vector Provider

The Vector provider enables semantic search using embeddings:

```typescript
import { createSupabaseVectorProvider } from "@axiomkit/supabase";

const vectorProvider = createSupabaseVectorProvider({
  url: "https://your-project.supabase.co",
  key: "your-supabase-anon-key",
  tableName: "vector_store", // optional
  embeddingDimension: 1536, // optional, default: 1536
});

await vectorProvider.initialize();

// Index documents with embeddings
await vectorProvider.index([
  {
    id: "doc1",
    content: "Machine learning is a subset of artificial intelligence",
    embedding: [0.1, 0.2, 0.3, ...], // 1536-dimensional vector
    metadata: { category: "AI", source: "wikipedia" }
  },
  {
    id: "doc2", 
    content: "Deep learning uses neural networks with multiple layers",
    embedding: [0.4, 0.5, 0.6, ...],
    metadata: { category: "AI", source: "research_paper" }
  }
]);

// Search for similar documents
const results = await vectorProvider.search({
  embedding: queryEmbedding,
  limit: 5,
  threshold: 0.7,
  filter: { category: "AI" }
});

// Results include similarity scores
console.log(results);
// [
//   { id: "doc1", content: "...", similarity: 0.85, metadata: {...} },
//   { id: "doc2", content: "...", similarity: 0.78, metadata: {...} }
// ]
```

### Graph Provider

The Graph provider manages relationships between entities:

```typescript
import { createSupabaseGraphProvider } from "@axiomkit/supabase";

const graphProvider = createSupabaseGraphProvider({
  url: "https://your-project.supabase.co",
  key: "your-supabase-anon-key",
  nodesTableName: "graph_nodes", // optional
  edgesTableName: "graph_edges", // optional
});

await graphProvider.initialize();

// Add nodes
const userNodeId = await graphProvider.addNode({
  id: "user:123",
  type: "user",
  properties: { name: "John Doe", email: "john@example.com" },
  labels: ["customer", "premium"]
});

const productNodeId = await graphProvider.addNode({
  id: "product:456",
  type: "product",
  properties: { name: "AI Assistant", price: 99.99 },
  labels: ["software", "ai"]
});

// Create relationships
await graphProvider.addEdge({
  sourceId: "user:123",
  targetId: "product:456",
  type: "purchased",
  properties: { date: "2024-01-15", amount: 99.99 }
});

// Query relationships
const purchases = await graphProvider.query({
  sourceId: "user:123",
  edgeType: "purchased"
});

// Traverse the graph
const path = await graphProvider.traverse({
  startNodeId: "user:123",
  maxDepth: 2,
  edgeTypes: ["purchased", "recommended"]
});
```

## Complete Memory System

The `createSupabaseMemory` function creates a complete memory system with all three providers:

```typescript
import { createSupabaseMemory } from "@axiomkit/supabase";
import { Agent } from "@axiomkit/core";

// Create the memory system
const memory = createSupabaseMemory({
  url: process.env.SUPABASE_URL!,
  key: process.env.SUPABASE_ANON_KEY!,
  kvTableName: "agent_kv_store",
  vectorTableName: "agent_vector_store", 
  nodesTableName: "agent_graph_nodes",
  edgesTableName: "agent_graph_edges",
  embeddingDimension: 1536,
});

// Initialize all providers
await memory.initialize();

// Use with an agent
const agent = new Agent({
  memory,
  llm: new OpenAIProvider({ apiKey: process.env.OPENAI_API_KEY }),
  prompt: "You are a helpful AI assistant with memory.",
});

// The agent now has access to all memory types
await agent.run("Remember that I prefer dark mode");
await agent.run("What was my preference again?");
```

### Error Handling

The providers include comprehensive error handling:

```typescript
try {
  await memory.initialize();
} catch (error) {
  if (error.code === 'PGRST116') {
    console.error('Database connection failed. Check your Supabase URL and key.');
  } else if (error.code === 'PGRST301') {
    console.error('Table not found. Run the setup SQL first.');
  } else {
    console.error('Unexpected error:', error.message);
  }
}

// Handle provider-specific errors
try {
  await memory.providers.vector.search(query);
} catch (error) {
  if (error.message.includes('pgvector')) {
    console.error('pgvector extension not enabled. Enable it in your Supabase dashboard.');
  }
}
```

## Performance Considerations

### Vector Search Optimization

For large vector datasets, consider these optimizations:

```sql
-- Create an optimized index for vector similarity search
CREATE INDEX IF NOT EXISTS idx_vector_store_embedding_hnsw 
ON vector_store USING hnsw (embedding vector_cosine_ops);

-- Set appropriate index parameters
ALTER INDEX idx_vector_store_embedding_hnsw SET (m = 16, ef_construction = 64);
```

### Connection Pooling

For high-traffic applications, consider connection pooling:

```typescript
import { createClient } from '@supabase/supabase-js';

// Create a client with connection pooling
const supabaseClient = createClient(url, key, {
  db: {
    pool: {
      min: 2,
      max: 10,
    },
  },
});
```

### Batch Operations

Use batch operations for better performance:

```typescript
// Batch vector indexing
const documents = [/* large array of documents */];
await memory.providers.vector.index(documents);

// Batch graph operations
const nodes = [/* array of nodes */];
const edges = [/* array of edges */];

await Promise.all([
  ...nodes.map(node => memory.providers.graph.addNode(node)),
  ...edges.map(edge => memory.providers.graph.addEdge(edge))
]);
```

## Security Best Practices

### Row Level Security (RLS)

Enable RLS policies for multi-tenant applications:

```sql
-- Enable RLS on tables
ALTER TABLE kv_store ENABLE ROW LEVEL SECURITY;
ALTER TABLE vector_store ENABLE ROW LEVEL SECURITY;
ALTER TABLE graph_nodes ENABLE ROW LEVEL SECURITY;
ALTER TABLE graph_edges ENABLE ROW LEVEL SECURITY;

-- Create policies (example for user-based access)
CREATE POLICY "Users can only access their own data" ON kv_store
  FOR ALL USING (key LIKE 'user:' || auth.uid() || ':%');

CREATE POLICY "Users can only access their own vectors" ON vector_store
  FOR ALL USING (metadata->>'userId' = auth.uid()::text);
```

## Troubleshooting

### Common Issues

**pgvector extension not found**
```bash
# Enable the pgvector extension in Supabase dashboard
# Go to Database > Extensions and enable 'vector'
```

**Table does not exist**
```bash
# Run the setup SQL in your Supabase SQL editor
# See the Database Setup section above
```

**Connection timeout**
```typescript
// Increase timeout in your Supabase client
const memory = createSupabaseMemory({
  url: process.env.SUPABASE_URL!,
  key: process.env.SUPABASE_ANON_KEY!,
  // Add timeout configuration if needed
});
```

**Vector dimension mismatch**
```typescript
// Ensure embedding dimension matches your model
const memory = createSupabaseMemory({
  url: process.env.SUPABASE_URL!,
  key: process.env.SUPABASE_ANON_KEY!,
  embeddingDimension: 768, // For models like BERT
  // or
  embeddingDimension: 1536, // For OpenAI embeddings
});
```

// Set environment variable
